## 🎯 목표 및 개요

* **목표:** 수업에서 배운 신경망 방법을 이용하여 **우리들의 신경망 모델**을 만든다.
* **핵심 가치:** 일정 관리, 동료와 협업 및 배려, 성취감 함양.
* **개요:**
    * 팀별로 협력하여 주어진 연구를 수행.
    * **Fashion-MNIST** 데이터셋에 대해 최선의 분류 성능을 내는 신경망 모델 구축.
    * 체계적이고 효과적인 실험을 통해 팀에서 제안하는 방법론의 합리성 입증.

---

## 🔬 연구 내용 및 제한 사항

### 1. 데이터셋: Fashion-MNIST

* **다운로드:** [https://github.com/zalandoresearch/fashion-mnist](https://github.com/zalandoresearch/fashion-mnist)
* **특징:** 28×28 흑백 이미지, 10개 카테고리, 총 70,000개 (학습 60,000개, 테스트 10,000개).

### 2. 방법론 제안

* **활용 범위:** 교재 **4, 5, 6, 7, 8장**의 내용만 활용.
* **제출:** 팀별로 Scratch로 만든 **최선의 모델 하나** 제안, PyTorch는 프레임워크의 편리함과 구조 확인용

### 3. 실험

* **필수 사항:** 효과적인 **실험 디자인** 및 **비교 실험**을 통해 방법의 합리성을 입증.
* **결과 정리:** 효과적인 결과 정리 필수.
* **데이터 사용:** 학습 시 **학습 데이터**만 활용; 테스트 데이터는 **성능 평가 때만** 사용.

### 4. Competition

* **제출:** 발표 당일, **학습된 모델 (코드)** 제출.
* **평가:** 교수자 컴퓨터에서 테스트 셋에 대한 **Accuracy**를 측정.
* **참고:** 제출 형식 추후 논의.

## ⬆️ Git 커밋규칙 (최종 모델 선정 이후부터 적용)
한 커밋당 변경사항은 무조건 한가지만!
| 타입 | 이름 | 내용 |
| :--- | :--- | :--- |
| **feat** | 새로운 기능 | 새로운 기능 추가에 대한 커밋 |
| **fix** | 버그 수정 | 버그 수정에 대한 커밋 |
| **build** | 빌드 시스템 | 빌드 관련 파일 수정 / 모듈 설치 또는 삭제에 대한 커밋 |
| **chore** | 기타 자잘한 수정 | 그 외 자잘한 수정에 대한 커밋 |
| **docs** | 문서 수정 | 문서 수정에 대한 커밋 |
| **style** | 코드 스타일 | 코드 스타일 혹은 포맷 등에 관한 커밋 (코드 내용 변경 X) |
| **refactor** | 리팩토링 | 코드 리팩토링 (기능 변경 X)에 대한 커밋 |
| **test** | 테스트 코드 | 테스트 코드 수정에 대한 커밋 |
| **perf** | 성능 개선 | 성능 개선에 대한 커밋 |

> 예시: 만약 `readme.md`를 수정했다면
